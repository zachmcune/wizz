# Data schema

All game content lives in `/data` as JSON, validated at load by Zod (`src/data/schemas.ts`).
Editing JSON changes the game with no code edits. Art/audio are referenced by key, so
swapping placeholder shapes for real sprites later is a data/provider change, not gameplay code.

- `/data/units/*.json` — `UnitDef` (cost, hp, armor, speed, sight, radius, weapon, producedBy, requires, art).
- `/data/buildings/*.json` — `BuildingDef` (cost, hp, footprint, requires, producesUnits, power, weapon, flags, art).
- `/data/spells/*.json` — `SpellDef` (cooldownTicks, targeting, effect: damage | buff | blink, requires).
- `/data/projectiles/*.json` — `ProjectileDef` (speed, art).
- `/data/maps/*.json` — `MapData` (tileW/tileH, tiles[], startLocations[], manaNodes[]). Generated by `scripts/gen-map.mjs`.
- `/data/match/*.json` — `MatchConfig` (mapId, seed, players[] with controller/team/color/startIndex/aiDifficulty).

Armor classes: `light | heavy | building`. Weapons carry a `vs` multiplier per armor class — this is the counter system.

Types live in `src/data/defs.ts`; the loader builds a `Registry` (`src/data/registry.ts`).
Loading fails loudly with the offending file + field path if any file is invalid.
